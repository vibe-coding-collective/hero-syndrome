# Prototype Dial Audio

This folder is only for local demo audio used by the disk UI prototype.

For now, use MP3 files only:

- `day-demo.mp3`
- `night-demo.mp3`

The prototype loads them from:

- `/prototype-dial/audio/day-demo.mp3`
- `/prototype-dial/audio/night-demo.mp3`

Keep local demo clips short while testing so the loading state, progress ring,
stop button, and Orb playback state are easy to inspect.

Do not commit generated or uploaded demo tracks unless the team explicitly
decides to keep sample media in Git. The final app should not depend on files in
this folder. Production music will be generated by ElevenLabs, stored by the
backend, and returned to the client as playable audio URLs or signed storage
URLs attached to a composition/session record.

AI agent notes:

- Treat this folder as a temporary local playback shim.
- Do not add API keys, user recordings, generated ElevenLabs tracks, or private
  user audio here.
- If the prototype needs different demo tracks, replace the local MP3 files with
  the same names instead of changing the public URLs.
- WAV support may be added later if the playback pipeline needs it, but the
  current prototype contract is MP3-only.
